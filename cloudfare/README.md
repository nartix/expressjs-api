# Cloudflare Containers Deployment

This directory contains the Cloudflare-only deployment surface for the ExpressJS API.

- `wrangler.jsonc` defines production and staging Worker environments, the Container image, the Durable Object binding, and the production custom domain.
- `src/index.ts` routes Worker requests into the Express container on port `5000` and injects Cloudflare Worker secrets as container environment variables.
- `worker-configuration.d.ts` is generated by Wrangler and provides the Worker runtime types used by TypeScript.
- `package.json` keeps the Wrangler dependencies and commands isolated from the existing API runtime package.

The existing Docker Compose and ECR deployment files are intentionally left untouched. They can still be used for local or future Docker deployment.

## Staging

The staging environment deploys to `workers.dev`, not to `expressjs.ferozfaiz.com`, so it is safe to test before production.

```bash
cd cloudfare
npm install --no-package-lock
npm run deploy:staging
```

The staging URL will look like:

```text
https://expressjs-api-worker-staging.<your-workers-subdomain>.workers.dev
```

## Production

Production disables `workers.dev` and serves the API from the custom domain `expressjs.ferozfaiz.com`.

```bash
cd cloudfare
npm install --no-package-lock
npm run deploy:production
```

## Worker Secrets

Create these Worker secrets for each Cloudflare environment before deploying:

- `pg_master_host`
- `pg_master_port`
- `pg_master_user`
- `pg_master_password`
- `pg_master_database`

To sync them from the existing repo `.env` file:

```bash
cd cloudfare
npm run secrets:sync:staging
```

For production:

```bash
cd cloudfare
npm run secrets:sync:production
```

To read a different env file, set `CLOUDFLARE_SECRETS_ENV_FILE`:

```bash
CLOUDFLARE_SECRETS_ENV_FILE=/path/to/.env.production npm run secrets:sync:production
```

To preview what would be synced without writing secrets:

```bash
DRY_RUN=true npm run secrets:sync:staging
```

You can still set secrets manually if needed:

```bash
cd cloudfare
wrangler secret put pg_master_host --config wrangler.jsonc
wrangler secret put pg_master_port --config wrangler.jsonc
wrangler secret put pg_master_user --config wrangler.jsonc
wrangler secret put pg_master_password --config wrangler.jsonc
wrangler secret put pg_master_database --config wrangler.jsonc
```

For staging, add `--env staging` to each `wrangler secret put` command.

## Useful Commands

```bash
npm run dev
npm run deploy:staging
npm run deploy:production
npm run containers:list
npm run containers:images
npm run types
```

Run `npm run types` again after changing `wrangler.jsonc`.

## Required Cloudflare Credentials

Set these environment variables locally or as CI secrets before deploying:

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

The API token needs permissions to deploy Workers, Containers, Durable Objects, and the `expressjs.ferozfaiz.com` Worker custom domain.

## DNS

`wrangler.jsonc` attaches the production Worker as a custom domain at `expressjs.ferozfaiz.com`. The hostname must be in a Cloudflare-managed zone and cannot already have a conflicting DNS record.

## Database Access

The deployed container reads the same PostgreSQL environment variables used by the existing Docker deployment. Make sure the database accepts connections from the Cloudflare Containers runtime before promoting production traffic.
